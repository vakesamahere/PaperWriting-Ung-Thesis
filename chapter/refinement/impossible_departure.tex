\subsection{不可能出发}
松弛模型可能利用物理上尚未到达的资源单位，其原因在于粗粒度时间映射可能使某些服务到达在聚合网络中过早可用。当我们试图在完整网络中实现该解时，这导致不可行的服务连接。
为检测不现实的服务连接，我们在时空网络中每个活跃节点$(i, t)$处应用流平衡检查。

在特定节点$(i, t)$处，来自上游的入流必须支持出流。我们将资源类型$f$的入弧分为几个组成部分。记$\holdingArcIn(i, t, f)$为资源类型$f$流入节点$(i, t)$的持有流，$\flightArc[-,legal](i, t, f)$为具有可行周转时间的服务，即弧$a = (i, t', j, t)$其中$t' + \travelTime_a \le t$，$\flightArc[-,illegal](i, t, f)$为$t' + \travelTime_a > t$的短弧。我们有
$\inArcSet{i,t}=\holdingArcIn(i, t, f)\uplus \flightArc[-,legal](i, t, f) \uplus \flightArc[-,illegal](i, t, f)$

节点$(i, t)$处资源类型$f$的出流由服务出发和持有流组成。记$\departArc(i, t, f)$为节点$(i, t)$处资源类型$f$的出发流，$\holdingArcOut(i, t, f)$为节点$(i, t)$处资源类型$f$的持有出流。我们有
$\outArcSet{i,t}=\departArc(i, t, f)\uplus \holdingArcOut(i, t, f)$

在任何节点$(i, t)$处，资源类型$f$可用于出发的资源单位是持有流和合法服务到达的总和，即$\sum_{a\in\holdingArcIn(i, t, f)}\fleetAssignVar_{a,f} + \sum_{a\in\flightArc[-,legal](i, t, f)}\fleetAssignVar_{a,f}$。实际出发需求为$\sum_{a\in\departArc(i, t, f)}\fleetAssignVar_{a,f}$。如果出发需求超过可用资源单位，即
\begin{equation}
    \sum_{a\in\departArc(i, t, f)}\fleetAssignVar_{a,f} > \sum_{a\in\holdingArcIn(i, t, f)}\fleetAssignVar_{a,f} + \sum_{a\in\flightArc[-,legal](i, t, f)}\fleetAssignVar_{a,f}
\end{equation}
则在节点$(i, t)$处资源类型$f$存在不可能出发。这意味着模型依赖非法服务弧来满足出发需求，这在现实中是不可行的。
对于每个活跃节点$(i, t)$和资源类型$f$，我们计算非法出发量$\impDeparts(i,t,f)$：
\begin{equation}
    \impDeparts(i,t,f) = \sum_{a\in\departArc(i, t, f)}\fleetAssignVar_{a,f} - \sum_{a\in\holdingArcIn(i, t, f)}\fleetAssignVar_{a,f} - \sum_{a\in\flightArc[-,legal](i, t, f)}\fleetAssignVar_{a,f}
\end{equation}

此处，$\impDeparts(i,t,f)$表示无法通过合法手段满足的资源单位净需求。如果$\impDeparts(i,t,f) > 0$，则模型被迫使用非法到达$\flightArc[-,illegal]$来满足此出发需求。该度量直接量化了非法性。

通过向离散化$\discretization$添加时间点，我们可以确保所有这些非法出发在更新后的离散化中不会发生。
