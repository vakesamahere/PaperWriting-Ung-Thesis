\subsection{完整网络上的上界修复模型}
\label{sec:ub}

下界模型在聚合网络上给出有效下界，但其解在映射回精确运营时间时往往物理不可行。为获得原问题的可行解，本节提出一个始终建立在完整时空网络$\full{\net}$上的MIP修复模型。该模型利用下界解提供的宏观结构信息——服务频率模式和超级行程收入上界——来引导完整网络上的排班修复，从而高效地产生高质量可行解作为上界。

\subsection{频率固定约束}
频率固定的核心假设是，虽然松弛解定义在粗糙网络上——因此当映射到精确时间时可能物理不可行——但它在资源分配方面做出了高度准确的决策。因此，松弛模型确定的资源频率是高度可信的，可以在当前迭代中锁定。

令$\fleetAssignVar^{*}$为从松弛模型获得的整数最优解。我们计算每个航段$s=(i,j)$和资源类型$f$的服务频率：
\begin{equation}
    \freq_{s,f} = \sum_{a \in \segArcSet{s}} \fleetAssignVar_{a,f}^{*}  \quad \in \mathbb{Z}_{\ge 0}
\end{equation}

然后我们通过强制执行这些计数来构造上界模型。我们将原始模型$\model{\discretizationFull}$中的通用频率约束~\eqref{constr:main.seg_freq}替换为：
\begin{equation}
    \sum_{a \in \full{\segArcSet{s}}} \fleetAssignVar_{a,f} = \freq_{s,f}, \quad \forall s \in \segSet, f \in \fleetSet
    % temp-command \tag{3e}
    \label{constr:ubm.freq_fix}
\end{equation}

通过强制求解器尊重松弛解的量，求解器的负担被减少为找到满足流平衡的最佳可行时间。这有效地将多方面的原始模型简化为纯调度问题。

\subsection{通过行程割解决收入高估}
标准频率固定仅在物理网络层面锁定了宏观资源分布，但留下了使模型难以求解的行程选择与收入高估问题。为了在完整网络上高效恢复可行解，我们将超级行程作为连接聚合空间与完整空间的结构化桥梁注入上界模型。我们引入连续市场份额变量$\shareSVar_{R,p,\ell} \in [0, 1]$，表示旅客类型$p$在票价等级$\ell$下选择超级行程$R$的市场需求比例。通过添加与超级行程绑定的行程割，我们强制完整网络上的细粒度行程组合必须受到其对应的超级行程收入上界的支配。这并非单纯的数值约束，而是利用聚合模型诱导的行程结构来限制完整空间中的搜索，从而显著收紧上界并加速分支定界过程的收敛。这些割的详细公式如下：
\begin{equation}
    \outAttr_{mp}\, \shareSVar_{Rp\ell} \;\le\; \attrSuper_{Rp\ell}\, \outsideShareVar_{mp}
    \quad \forall m\in \marketSet,\, \forall p\in \psgTypeSet,\, \forall R\in \itinSuperInMarket_{m},\, \forall \ell\in \fareClassSet
    % temp-command \tag{3f}
    \label{constr:ubm.itinerary_cut.choice.ratio}
\end{equation}
\begin{equation}
    \left(\frac{\adjOutAttr_{mp}}{\outAttr_{mp}}\right) \outsideShareVar_{mp}
    \;+\;
    \sum_{R\in \itinSuperInMarket_{m}}\sum_{\ell\in \fareClassSet} 
    \left(\frac{\adjAttrSuper_{Rp\ell}}{\attrSuper_{Rp\ell}}\right) \shareSVar_{Rp\ell}
    \;\le\; 1
    \quad \forall m\in \marketSet,\; \forall p\in \psgTypeSet
    % temp-command \tag{3g}
    \label{constr:ubm.itinerary_cut.choice.limit}
\end{equation}
\begin{equation}
    \sum_{m\in \marketSet}\sum_{p\in \psgTypeSet}\sum_{\ell\in \fareClassSet}\sum_{R\in \itinSuperUsingArc{m}{a}} 
    \demand_{mp}\, \shareSVar_{Rp\ell}\;\le\; \sum_{a'\in \Phi^{-1}(a)}\sum_{f\in \fleetSet} \fleetCap_{f}\, \fleetAssignVar_{a'f}
    \quad \forall a\in \flightArc
    % temp-command \tag{3h}
    \label{constr:ubm.itinerary_cut.choice.capacity}
\end{equation}
\begin{equation}
    \sum_{a'\in \Phi^{-1}(a)}\sum_{f\in \fleetSet} \fleetAssignVar_{a'f} \;\ge\; \shareSVar_{Rp\ell}
    \quad \forall a\in \flightArc,\; \forall m\in \marketSet,\, \forall R\in \itinSuperUsingArc{m}{a},\, \forall p\in \psgTypeSet,\,\forall \ell\in\fareClassSet
    % temp-command \tag{3i}
    \label{constr:ubm.itinerary_cut.choice.covering}
\end{equation}
\begin{equation}
   \shareSVar_{R,p,\ell} \cdot \fare_{R,\ell} \;\le\; \sum_{r \in R} \shareVar_{r, p, \ell} \cdot \fare_{r,\ell}
   % temp-command \tag{3n}
   \label{constr:ubm.itinerary_cut.cut}
\end{equation}

总体而言，约束\eqref{constr:ubm.itinerary_cut.choice.ratio}-\eqref{constr:ubm.itinerary_cut.choice.covering}确保超级行程份额$\shareSVar_{R,p,\ell}$在宏观层面（部分网络中的时间窗口）表示客户选择行为，并产生不少于其组成细粒度行程聚合份额的份额。割\eqref{constr:ubm.itinerary_cut.cut}明确将超级行程份额变量与细粒度份额变量联系起来，确保超级行程的收入贡献支配其底层行程的总和。通过强制超级行程与其组成行程之间的这些关系，我们有效地收紧了上界并引导求解器朝向解空间中更有前景的区域，从而加速收敛到全局最优。

令$\textbf{c}^{\mathrm{LB}}$表示$\model{\discretization, \itinSetSuper}$最优目标值的成本项。令$\textbf{r}^{\mathrm{LB}}$表示$\model{\discretization, \itinSetSuper}$最优目标值的收入项。由命题~\ref{pro.valid_relaxation}，$\model{\discretization, \itinSetSuper}$构成完整模型的有效松弛；由此可得原始问题任何可行解的最优收入以$\textbf{r}^{\mathrm{LB}}$为下界。我们通过附加以下有效不等式来明确利用这一界：
\begin{equation}
    \sum_{m\in \marketSet}\sum_{p\in \psgTypeSet}\sum_{\ell\in \fareClassSet}\sum_{r\in \itinInMarket_m}
    \demand_{mp}\, \fare_{r\ell}\, \shareVar_{rp\ell} \;\le\; \textbf{r}^{\mathrm{LB}}
    % temp-command \tag{3o}
    \label{constr:ubm.obj_cut}
\end{equation}
该割明确排除了总收入严格低于已知下界的所有LP解，从而收紧可行域并减少分支定界过程的有效搜索空间。

因此，我们有一个协同结合频率固定和行程割的上界生成模块，有效地将松弛LB解修复为高质量可行解，引导精确求解器朝向全局最优，写作如下：
\begin{equation}\begin{aligned}
    \ubm(\discretizationFull,\model{\discretization, \itinSetSuper}) = \min_{\shareSVar, \shareVar, \fleetAssignVar} \quad & \textbf{c}^{\mathrm{LB}} - \sum_{m\in \marketSet}\sum_{p\in \psgTypeSet}\sum_{\ell\in \fareClassSet}\sum_{r\in \itinInMarket_m} \demand_{mp}\, \shareVar_{rp\ell} \fare_{r,\ell}
    % temp-command \tag{3a}
\\
    \text{s.t.} \quad
    &
    \eqref{constr:main.flow}, \eqref{constr:main.fleet_available}, \eqref{constr:main.cooldown}, \eqref{constr:main.seg_freq} \notag
\\&
    \eqref{constr:main.choice.ratio}, \eqref{constr:main.choice.limit}, \eqref{constr:main.choice.capacity}, \eqref{constr:main.choice.covering} \notag
\\&
    \eqref{constr:main.domain.x.flight}, \eqref{constr:main.domain.x.holding}, \eqref{constr:main.domain.w.out}, \eqref{constr:main.domain.w} \notag
\\&
    \eqref{constr:ubm.freq_fix}, \eqref{constr:ubm.itinerary_cut.choice.ratio}, \eqref{constr:ubm.itinerary_cut.choice.limit}, \eqref{constr:ubm.itinerary_cut.choice.capacity}, \eqref{constr:ubm.itinerary_cut.choice.covering}, \eqref{constr:ubm.itinerary_cut.cut}, \eqref{constr:ubm.obj_cut} \notag
\end{aligned}\end{equation}

\subsection{变量缩减}
为加速求解过程，我们基于固定的聚合频率$\freq_{s,f}$应用一系列精确的变量缩减。由于$\freq_{s,f}$决定了可用网络容量，任何零分配服务的航段从根本上破坏了连通性，使我们能够安全地将许多变量固定为零，并在将模型发送给求解器之前消除冗余的下游变量和约束。

对于任何航段$s \in \segSet$和资源类型$f \in \fleetSet$，如果锁定频率$\freq_{s,f} = 0$，则意味着不能在航段$s$上运营$f$型服务。因此，我们立即将此零分配级联到所有底层服务分配变量：
\begin{equation}
    \fleetAssignVar_{a,f} = 0, \quad \forall a \in \full{\segArcSet{s}} \text{ if } \freq_{s,f} = 0
\end{equation}
这对这些组合内在地满足频率约束\eqref{constr:ubm.freq_fix}，使我们能够从域中移除这些变量和约束。

由于行程$r$（或超级行程$R$）从根本上由一系列时空服务弧组成，移除其任何底层空间航段的所有容量直接切断了这些物理路径。为记号方便，我们写$s \in r$（或$s \in R$）表示空间航段$s$被行程$r$（或超级行程$R$）的时空弧所经过。如果行程依赖于没有任何资源类型分配服务的航段$s$（即$\sum_{f\in\fleetSet} \freq_{s,f} = 0$），则该行程变得物理不可能。由于覆盖约束\eqref{constr:main.choice.covering}和\eqref{constr:ubm.itinerary_cut.choice.covering}，与此类行程相关的市场份额降为零：
\begin{equation}
    \shareVar_{r,p,\ell} = 0, \quad \text{if } \exists s \in r \text{ s.t. } \sum_{f\in\fleetSet} \freq_{s,f} = 0
\end{equation}
\begin{equation}
    \shareSVar_{R,p,\ell} = 0, \quad \text{if } \exists s \in R \text{ s.t. } \sum_{f\in\fleetSet} \freq_{s,f} = 0
\end{equation}
这些零值变量可以从模型中明确移除。因此，给定超级行程$R$的行程割约束\eqref{constr:ubm.itinerary_cut.cut}只有在其所有组成细粒度行程$r \in R$都被关闭（即固定为零）时才能完全删除。如果只有$r \in R$的子集被移除，约束\eqref{constr:ubm.itinerary_cut.cut}必须保留以将超级行程变量$sw_R$与剩余活跃的细粒度行程绑定。

对于任何市场$m \in \marketSet$，如果其所有组成活跃行程$r \in \itinInMarket_{m}$和超级行程$R \in \itinSuperInMarket_{m}$都被上述缩减消除，则触发纯未服务场景。该市场中的旅客根本没有可选择的可用服务选项。
根据选择限制恒等约束\eqref{constr:main.choice.limit}，外部选项份额变量$\outsideShareVar_{mp}$不可避免地变为完全由相应效用限制的确定性常数：
\begin{equation}
    \outsideShareVar_{mp} = \frac{\outAttr_{mp}}{\adjOutAttr_{mp}}, \quad \text{if } \shareVar_{r,p,\ell} = 0 \; \forall r \in \itinInMarket_m
\end{equation}
因此，我们可以将$\outsideShareVar_{mp}$作为变量完全消除，并将市场$m$相关的选择限制约束\eqref{constr:main.choice.limit}和比率限制约束\eqref{constr:main.choice.ratio}从优化框架中完全移除，显著压缩约束矩阵维度。
